Abstract

Advanced technologies have dramatically changed the ways that accounting professionals collect, analyze and interpret data. Embracing the prevalent implementation of advanced technologies, accounting is moving towards a STEM discipline and requesting accounting students be equipped with emerging technologies.

In response to the increasing requirements from the Association to Advance Collegiate Schools of Business (AACSB) and CPA exams on students' data analytics skills, this case provides you an opportunity to complete the data analytics process by analyzing Dillard's data using Structured Query Language (SQL) and Tableau.

Specifically, this case provides an illustration of real-world data analysis, including the use of SQL to extract required data, the use of word clouds, geographic maps, and box-and-whisker plots to visualize data and outliers, an illustration of grouping data by clustering, and using interactive dashboards and narrative stories to communicate results.

Research Objective

We aim to use software tools that you are familiar with to inspire your interest in data analytics, helping you understand the enormous power of SQL and Tableau to query, aggregate, analyze and visualize massive data as you build your own data analytics mindset.

1. The Case

About Dillard's Inc.

Dillard's, Inc. is a high-end department store chain based in Little Rock, Arkansas. It offers one-stop shopping for clothes, shoes, bags, accessories, home and beauty products.

Founded

1938 by William T Dillard

Store Count

282 stores across 29 states

Public Since

1969 on American Stock Exchange

Top Market

Texas with 57 establishments

Your Role in This Case

Position: Manager of the Cosmetics Department at Dillard's

Task: The company's general manager has requested a comprehensive sales performance report for the Christmas season (December 2015 to January 2016)

Objective: Analyze cosmetic department performance, identify trends, and provide actionable business insights

Database Structure

You will work with Dillard's transactional database containing the following key tables:

Figure 1: UML diagram showing the relationship between TRANSACT, SKU, and DEPARTMENT tables

TRANSACT Table

  • TRAN_AMT - Transaction Amount
  • ORIG_PRICE - Original Price
  • SALE_PRICE - Sale Price
  • TRAN_DATE - Transaction Date
  • COST - Product Cost
  • STATE - Store State
  • TRAN_TYPE - Transaction Type
  • TENDER_TYPE - Payment Method

SKU Table

  • SKU - Stock Keeping Unit ID
  • BRAND - Product Brand
  • SIZE - Product Size
  • COLOR - Product Color
  • STYLE - Product Style

DEPARTMENT Table

  • DEPT - Department Code
  • DEPT_DESC - Department Description
  • DEPTCENT - Department Center
  • DEPTCENT_DESC - Department Center Description

2. Deliverables

You need to prepare a presentation to communicate your analysis with the CEO and other senior managers. The static report is required to be submitted before the presentation.

Analysis Report

Format: 12-point Times New Roman font, single-spaced, one-inch margins

Content: Summarizing key findings including:

  • Analysis of discount levels across different states
  • Trend of discount levels over weekdays
  • Popular cosmetic brands analysis
  • Natural clusters of cosmetic brands based on profit margin and total sales

Required Visualizations

Create the following specific visualizations:

  • Geographic Map: State average discount rates for Christmas season
  • Word Cloud: Total sales of each cosmetic brand
  • Box-and-Whisker Plots: Distribution of discount rates per weekday
  • Clustering Analysis: Natural clusters of cosmetic brands
  • Trend Analysis: Sales growth rates over time

SQL Queries

Extract data using SQL queries to obtain:

  • Transaction amounts, original prices, sale prices
  • Transaction dates and cost information
  • Department descriptions and state information
  • Transaction types and tender types

Sample Query Structure:

SELECT TRAN_AMT, ORIG_PRICE, SALE_PRICE, TRAN_DATE, COST, 
       DEPT_DESC, STATE, DEPTCENT_DESC, TRAN_TYPE, TENDER_TYPE
FROM TRANSACT
INNER JOIN SKU ON TRANSACT.SKU = SKU.SKU
INNER JOIN DEPARTMENT ON SKU.DEPT = DEPARTMENT.DEPT
WHERE DEPT_DESC = 'COSMETICS'
  AND TRAN_DATE BETWEEN '2015-12-01' AND '2016-01-31';

Executive Presentation

Format: PowerPoint or Tableau Story

Content:

  • Interactive dashboard with word cloud as filter
  • Story highlighting insights from each graph
  • Clear narrative flow for executive audience
  • Actionable recommendations based on findings

Key Metrics to Calculate

Discount Rate

Discount Rate = (ORIG_PRICE - SALE_PRICE) / ORIG_PRICE × 100

Target: 5% benchmark for comparison

Profit Margin

Profit Margin = (TRAN_AMT - COST) / TRAN_AMT × 100

Analyze profitability by brand and region

Return Rate

Return Rate = (Return Transactions / Total Transactions) × 100

Compare against industry average of 5.21%

3. Teaching Note

3.1 Motivation

This case addresses the growing need for accounting students to develop data analytics skills in response to technological advances and industry demands.

  • AACSB accreditation requirements emphasize data analytics competencies
  • CPA exam evolution includes data analytics and technology components
  • Industry demand for accountants with technical and analytical skills
  • Integration of STEM principles into accounting education

3.3 Educational Objectives

SQL Proficiency

Develop skills in writing complex SQL queries, joining multiple tables, and extracting meaningful business data from large databases.

Data Visualization

Master Tableau for creating professional visualizations including maps, word clouds, box plots, and clustering analysis.

Analytical Thinking

Develop critical thinking skills to interpret data patterns, identify business insights, and make data-driven recommendations.

Business Communication

Learn to communicate complex analytical findings to executive audiences through clear visualizations and compelling narratives.

3.4 Case Prerequisites

Knowledge Requirements

  • Basic accounting principles and financial statement analysis
  • Fundamental understanding of retail business operations
  • Introduction to database concepts and relational data models

Technical Skills

  • Basic SQL query writing and database navigation
  • Familiarity with Tableau Desktop interface and basic chart creation
  • Microsoft Office proficiency (Excel, PowerPoint)

3.6 Implementation Guidance

1

Database Setup & Exploration (Week 1)

Students access the Dillard's database, explore table structures, and understand data relationships using the UML diagram.

2

SQL Query Development (Week 2)

Write and test SQL queries to extract required data, focusing on joins between TRANSACT, SKU, and DEPARTMENT tables.

3

Data Analysis & Visualization (Week 3-4)

Import data into Tableau, create required visualizations, and perform statistical analysis including clustering.

4

Report Writing & Presentation (Week 5)

Compile findings into executive report format and prepare presentation for management audience.
